## 问题分析
属性面板的开关属性（特别是"是否启用"属性）没有对齐组件列表的实际开启状态，主要原因是：

1. 组件列表的开关状态绑定到组件的 `enabled` 属性，该属性在加载时根据 `required` 属性设置
2. 而属性面板的开关状态使用的是属性定义中的 `defaultValue`，该值在 JSON 文件中固定为 `false`
3. 当组件被标记为 `required` 时，组件列表的开关会自动开启，但属性面板的开关仍然显示关闭状态

## 解决方案
修改 `Workbench.vue` 文件中的属性面板开关渲染逻辑，使"是否启用"开关（类型为 `component_switch`）与组件的实际 `enabled` 状态同步：

1. **更新模板中的开关渲染代码**：
   - 对于类型为 `component_switch` 的属性，绑定 `:checked` 到 `selectedComponent.enabled`
   - 添加 `:disabled` 属性，当组件被锁定时禁用开关
   - 添加 `@change` 事件监听器，在开关状态改变时更新组件的 `enabled` 状态

2. **保持其他类型开关的现有行为**：
   - 对于类型为 `switch` 的普通开关属性，继续使用 `property.defaultValue`

## 具体修改
在 `Workbench.vue` 文件中，找到属性面板开关渲染的代码部分，修改为：

```vue
<!-- 开关类型 -->
<label v-if="property.type === 'component_switch'" class="toggle-switch">
  <input 
    type="checkbox" 
    :checked="selectedComponent?.enabled || false"
    :disabled="selectedComponent?.locked || false"
    @change="e => {
      if (!selectedComponent?.locked) {
        selectedComponent.enabled = e.target.checked
        toggleComponent(selectedComponent)
      }
    }"
  >
  <span class="toggle-slider"></span>
</label>
<label v-else-if="property.type === 'switch'" class="toggle-switch">
  <input 
    type="checkbox" 
    :checked="property.defaultValue || false"
  >
  <span class="toggle-slider"></span>
</label>
```

这样修改后，属性面板的"是否启用"开关将与组件列表的实际开启状态保持一致，同时保持其他开关属性的现有行为。
