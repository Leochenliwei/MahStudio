# 表达式输入框细节优化 Plan

## 任务列表

### 任务 1: 输入框高度自适应与换行
**问题**: 输入框目前固定高度 40px，当运算元素较多时无法自动换行显示

**解决方案**:
1. 修改 `.input-trigger` 样式：
   - 将固定高度 `height: 40px` 改为最小高度 `min-height: 40px`
   - 添加 `flex-wrap: wrap` 允许内容换行
   - 添加 `padding-top` 和 `padding-bottom` 保证垂直间距

2. 修改 `.input-content` 样式：
   - 添加 `flex-wrap: wrap` 允许表达式单元换行
   - 调整 `gap` 保证换行后的间距

3. 设置最大高度限制：
   - 添加 `max-height: 120px`
   - 超出时显示滚动条

### 任务 2: 修复 IdentitySelector 层级问题
**问题**: 身份选择器 Dropdown 被元素选择器面板遮挡

**解决方案**:
1. 提升 IdentitySelector 的 z-index：
   - 将 `z-index: 9999` 改为 `z-index: 10001`

2. 或降低元素选择器面板的 z-index：
   - 将元素选择器的 `z-index: 9999` 改为 `z-index: 9998`

**推荐方案**: 提升 IdentitySelector 的 z-index 到 10001，确保它始终在最上层

### 任务 3: 数字和运算符添加浅灰底
**问题**: 数字和运算符目前显示为纯文本，需要作为运算元素添加浅灰底

**解决方案**:
1. 修改 `expressionUnits` computed：
   - 将连续的数字识别为一个 number 类型的单元
   - 将运算符（+、-、*、/、>、< 等）识别为 operator 类型的单元

2. 在模板中添加数字和运算符的渲染：
   - 数字：浅灰底 `#f3f4f6`，文字 `#374151`
   - 运算符：浅灰底 `#f3f4f6`，文字 `#6b7280`

3. 样式设计：
   - 圆角 4px
   - padding: 2px 6px
   - 与表达式单元保持 4px 的 gap

## 实现步骤

1. **任务 1**: 修改输入框样式，支持自适应高度和换行
2. **任务 2**: 调整 IdentitySelector 的 z-index
3. **任务 3**: 实现数字和运算符的识别与渲染

## 预期效果

1. 输入框可以根据内容自动增高，最多显示 3 行，超出显示滚动条
2. 身份选择器始终显示在最上层，不被其他面板遮挡
3. 数字和运算符显示为浅灰底的标签，与表达式单元区分开
