# 修改RoomCreator组件以还原HTML原型界面和交互逻辑

## 分析对比

### HTML原型文件分析
- **界面结构**：顶部tab栏（添加分组、选项联动按钮），中间内容区（基础参数行、动态分组容器）
- **交互逻辑**：
  - 点击人数/局数旁边的⚙️图标打开左侧抽屉设置
  - 点击分组内的添加元件按钮添加单选/复选/下拉列表/下拉&开关控件
  - 点击控件旁边的⚙️图标打开右侧抽屉进行属性配置
  - 点击规则标签跳转到高级规则页面

### 现有Vue组件分析
- **界面结构**：模态框形式，包含基础规则和高级规则两个标签页
- **交互逻辑**：
  - 基础规则标签页：显示人数、局数、底分、时间配置
  - 高级规则标签页：显示分组管理、控件添加和属性配置
  - 点击详细规则按钮跳转到高级规则页面

## 修改计划

### 1. 重构界面结构
- 将模态框改为全屏布局，与HTML原型保持一致
- 添加左侧侧边栏，包含添加分组和选项联动按钮
- 中间内容区改为类似HTML原型的布局，包含基础参数行和动态分组容器

### 2. 实现抽屉式设置面板
- 创建左侧抽屉用于人数和局数设置
- 创建右侧抽屉用于控件属性配置
- 实现抽屉的打开/关闭动画效果

### 3. 优化交互逻辑
- 修改基础参数行的显示和交互，添加⚙️图标按钮
- 实现分组的添加、管理功能
- 实现控件的添加、预览和属性编辑功能
- 优化规则标签的显示和点击跳转逻辑

### 4. 样式调整
- 调整整体布局和样式，与HTML原型保持一致
- 优化按钮、输入框、分组等元素的样式
- 添加适当的动画和过渡效果

### 5. 数据结构调整
- 确保数据结构与HTML原型的交互逻辑匹配
- 保持与现有localStorage数据持久化方案的兼容性

## 技术实现
- 使用Vue 3 Composition API
- 利用Vue的响应式系统管理状态
- 实现抽屉组件和相关动画
- 保持代码结构清晰，易于维护
