# 实现工作台画布区域的 Category 容器

## 1. 功能分析
- **需求**：在工作台画布区域添加多个 category 作为虚线容器，模拟流程流转过程
- **目标**：每个组件可以拖拽到对应的 category 中，实现流程化的组件组织

## 2. 实现方案

### 2.1 数据结构设计
- 添加 `categories` 数组，包含多个 category 对象
- 每个 category 包含：id、name、x、y、width、height、components 等属性

### 2.2 模板修改
- 在 `workbench-canvas` 中添加 category 容器
- 每个 category 显示为虚线框，带有标题
- 组件可以在 category 内部或之间拖拽

### 2.3 功能实现
- 实现组件拖拽到 category 中的逻辑
- 跟踪每个 category 中的组件
- 支持组件在不同 category 之间移动

### 2.4 样式优化
- 为 category 设计虚线容器样式
- 添加 category 选中状态
- 优化组件在 category 中的显示效果

## 3. 技术实现

### 3.1 数据管理
- 添加 `categories` 响应式数据
- 实现 `addCategory`、`removeCategory` 等方法
- 跟踪组件与 category 的关联关系

### 3.2 拖拽处理
- 修改 `onDrop` 事件处理，支持将组件放置到 category 中
- 实现 `onCategoryDrop` 方法，处理组件拖拽到 category 的逻辑
- 更新组件的位置和所属 category

### 3.3 视觉效果
- 为 category 添加虚线边框样式
- 实现 category 高亮效果
- 添加流程流转的视觉提示

## 4. 预期效果
- 画布区域显示多个虚线 category 容器
- 组件可以拖拽到任意 category 中
- 每个 category 显示其中的组件数量
- 支持 category 的添加、删除和调整大小
- 提供清晰的流程流转视觉效果
