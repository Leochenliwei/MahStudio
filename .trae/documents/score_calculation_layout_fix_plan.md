# 计分规则模块布局修复计划

## 问题分析
当前计分规则模块存在以下布局问题：
- 头部统计信息区域未固定，会随滚动而移动
- 底部添加场景按钮区域布局不稳定
- 中间场景卡片滚动区域需要优化

## 修复目标
- 固定显示头部（head）和底部（foot）
- 中间场景卡片区域作为独立的滑动区域
- 放大整体弹窗高度，增加滑动视窗大小
- 保持良好的视觉层次和交互体验

## 修复总结

### 已完成的修复
1. **放大整体弹窗高度**：将弹窗高度设置为 `90vh`，增大了滑动视窗大小
2. **调整整体布局结构**：使用 Flexbox 布局确保头部和底部固定，中间区域可滚动
3. **修复头部固定布局**：确保 `editor-header` 固定在顶部
4. **修复底部固定布局**：确保 `section-foot` 固定在计分规则区域底部
5. **优化中间滚动区域**：添加自定义滚动条样式，确保场景卡片区域可顺畅滚动
6. **确保场景卡片高度固定**：调整布局确保卡片高度由内容决定，不均分视窗高度

### 技术实现
- 使用 Flexbox 布局实现固定头部和底部
- 设置 `overflow-y: auto` 实现中间区域滚动
- 添加自定义滚动条样式提升用户体验
- 调整弹窗高度为 `90vh` 增加可视区域
- 确保场景卡片高度由内容决定，不随视窗高度均分
