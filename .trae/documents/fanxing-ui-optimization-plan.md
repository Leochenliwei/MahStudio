# 番型说明界面UI排版优化计划

## 当前问题分析

根据用户选择的元素分析，当前番型说明使用右侧抽屉（drawer）形式展示，存在以下问题：
1. 抽屉宽度仅30%，空间局促
2. 示例牌型展示为纯文本标签，不够直观
3. 信息层级不够清晰，描述和备注混杂
4. 整体视觉美感有待提升

## 优化方案

### 方案：卡片式弹窗 + 可视化牌型展示

将右侧抽屉改为居中卡片式弹窗，提供更充足的展示空间，并优化牌型展示方式。

## 实施步骤

### 1. 修改展示容器
- 将 `custom-drawer-overlay` 改为居中弹窗形式
- 增大展示区域，宽度从30%调整为500px固定宽度
- 添加圆角和阴影，符合设计规范

### 2. 优化信息层级
- 番型名称：使用大标题样式（18px, semibold）
- 描述区域：独立卡片展示，带图标标识
- 备注区域：独立卡片展示，使用浅色背景区分
- 示例牌型：使用可视化牌面展示替代纯文本

### 3. 可视化牌型展示
- 创建牌面组件样式，模拟真实麻将牌外观
- 万/条/筒：显示数字+花色
- 字牌：显示文字（东/南/西/北/中/发/白）
- 牌面添加轻微阴影和边框，增强立体感

### 4. 样式优化
- 按照 design_rule.md 规范使用颜色
- 主题色：#3b82f6
- 背景色：#ffffff
- 边框色：#e5e7eb
- 圆角：8px
- 间距：16px/24px

### 5. 动画效果
- 弹窗进入：scale + fade 组合动画
- 背景遮罩：fade 动画
- 关闭按钮：hover 颜色变化

## 代码修改清单

### 修改文件：FanXingDialog.vue

- 将 drawer 改为 modal 形式
- 调整容器类名和结构

- 重写 `.custom-drawer-overlay` 为居中布局
- 调整 `.custom-drawer` 宽度和样式
- 添加牌面样式 `.mahjong-card`
- 优化信息卡片样式

## 预期效果

1. 更充足的展示空间，信息不再拥挤
2. 清晰的视觉层级，用户快速获取关键信息
3. 直观的牌型展示，提升用户体验
4. 符合整体设计规范，视觉统一

- 仅修改展示层样式和结构
- 不影响其他组件功能
