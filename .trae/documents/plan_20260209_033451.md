# 从 JSON 文件中整理组件列表和属性

## 1. 任务分析
- **输入**：`/Users/zonst/Documents/Programs/WEBconfig/资料/rule_config fixlaizi.json` 文件
- **输出**：结构化的组件列表，其中每个 tabcontainer 作为一个组件，子节点作为该组件的属性

## 2. 实现步骤

### 2.1 解析 JSON 文件
- 读取并解析 JSON 文件内容
- 定位到根节点的 elements 数组

### 2.2 提取 tabcontainer 组件
- 遍历 elements 数组，筛选出 type 为 "tab_container" 的元素
- 为每个 tabcontainer 创建组件对象，包含以下信息：
  - id：组件唯一标识
  - name：组件名称（使用 title 字段）
  - category：组件分类
  - properties：组件属性列表

### 2.3 整理组件属性
- 对于每个 tabcontainer 的 elements 数组，将其中的每个元素作为组件的属性
- 每个属性包含：
  - id：属性唯一标识
  - name：属性名称（使用 title 字段）
  - type：属性类型
  - defaultValue：默认值（如果有）
  - 其他相关属性（如 datas、visibleWhen 等）

### 2.4 生成结果
- 将整理好的组件列表保存为新的 JSON 文件
- 确保结构清晰，便于后续使用

## 3. 技术实现
- 使用 JavaScript 编写脚本，读取和解析 JSON 文件
- 遍历和筛选数据，生成目标结构
- 将结果写入新文件

## 4. 预期输出
- 生成一个新的 JSON 文件，包含所有 tabcontainer 组件及其属性
- 每个组件包含完整的属性信息，便于后续的工作台使用
