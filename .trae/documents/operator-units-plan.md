# 运算符单元化改造计划

## 目标
将 ExpressionInput 组件中的所有运算符（+、-、*、/、>、<、>=、<=、&&、||、! 等）都作为独立的运算单元存在，而不是纯文本显示。

## 当前状态
- 运算符目前以纯文本形式显示在表达式中
- 表达式单元只有 `[身份.元素]` 这种游戏元素

## 改造方案

### 1. 定义运算符单元类型
```typescript
interface OperatorUnit {
  type: 'operator'
  value: string        // 原始运算符如 '+', '-', '&&'
  display: string      // 显示文本如 '+', '-', '且'
  category: 'arithmetic' | 'compare' | 'logic' | 'bracket'
}
```

### 2. 表达式解析改造
- 修改 `expressionUnits` 计算属性，同时解析游戏元素和运算符
- 使用正则表达式识别各类运算符
- 保持原有游戏元素单元的解析逻辑

### 3. 渲染改造
- 在模板中为运算符单元添加独立的渲染逻辑
- 运算符单元使用标签样式展示，与游戏元素区分
- 支持点击选中运算符单元

### 4. 键盘输入改造
- 点击虚拟键盘的运算符按钮时，插入运算符单元
- 支持在光标位置插入运算符

### 5. 样式设计
- 算术运算符（+、-、*、/）：蓝色主题
- 比较运算符（>、<、>=、<=）：橙色主题
- 逻辑运算符（&&、||、!）：绿色主题
- 括号（(、)）：灰色主题

## 实施步骤

### 步骤1：类型定义扩展
- 扩展 `ExpressionUnit` 类型，支持运算符单元
- 添加运算符分类常量

### 步骤2：表达式解析重构
- 重构 `expressionUnits` 计算属性
- 实现 `parseExpression` 方法，统一解析元素和运算符
- 实现 `parseOperators` 方法，专门解析运算符

### 步骤3：模板渲染改造
- 在 `v-for` 循环中添加运算符单元的渲染分支
- 为运算符单元添加点击选中功能
- 保持游戏元素单元的渲染不变

### 步骤4：键盘输入改造
- 修改 `insertKey` 方法，插入运算符时创建单元
- 修改 `removeLast` 方法，支持删除运算符单元

### 步骤5：光标定位优化
- 优化光标在运算符单元间的定位逻辑
- 支持键盘左右箭头在单元间移动

### 步骤6：样式实现
- 添加运算符单元的样式类
- 实现不同类别运算符的颜色主题

## 预期效果
- 表达式 `[cur_player.hand_count] + [winner.hu_count] > 10` 将被解析为：
  1. `[cur_player.hand_count]` - 游戏元素单元
  2. `+` - 算术运算符单元
  3. `[winner.hu_count]` - 游戏元素单元
  4. `>` - 比较运算符单元
  5. `10` - 数字单元
