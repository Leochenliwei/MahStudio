# 实现选中category组件时右侧属性栏显示组件属性

## 1. 添加选中category组件的逻辑
- 在 `category-component-item` 中添加 `@click` 事件，调用 `selectCategoryComponent` 函数
- 添加 `selectedCategoryComponent` 状态变量，用于存储当前选中的category组件
- 添加 `selectCategoryComponent` 函数，用于处理category组件的选中逻辑
- 添加 `clearSelection` 函数，用于清除选中状态

## 2. 修改右侧属性栏的逻辑
- 修改右侧属性栏的显示逻辑，当选中category组件时，显示该组件的属性
- 添加 `selectedComponent` 计算属性，用于获取当前选中的组件（无论是panel还是category组件）
- 添加 `getSelectedComponentProperties` 函数，用于获取当前选中组件的属性

## 3. 根据属性类型显示不同的元件
- 根据组件属性的 `type` 字段，显示不同的元件
- 支持的元件类型包括：
  - `component_switch` / `switch`：开关
  - `toggle`：单选按钮组
  - `select`：下拉选择框
  - `checkbox`：复选框
  - `button`：按钮
  - 其他类型：文本输入框

## 4. 使用正确的数据源
- 确保使用从 `components_list.json` 文件中获取的组件属性数据
- 确保属性的 `defaultValue`、`datas` 等字段被正确使用
- 确保属性的 `visibleWhen` 条件被正确处理

## 5. 样式优化
- 为属性栏中的不同元件添加适当的样式
- 确保属性栏的布局美观，响应式
- 优化属性编辑的用户体验
