# 算分公式编辑器 - 动态模式切换优化方案

## 需求分析

当前实现：使用标签页固定分为「多场景」和「单场景」两个标签页，用户手动切换

需求变更：
- **默认**：单场景模式（只有一个计分规则）
- **当用户点击「添加场景」按钮**：变成多场景模式（多个场景卡片列表）
- **当用户删除场景，仅剩一个时**：自动变回单场景模式
- 保持原有的数据结构和输出格式不变

## 设计思路

### 信息架构

```
赢家判定（固定）
└─ 计分规则（动态展示）
   ├─ 单场景模式：只有一套表单项（出分玩家 + 计分表达式）
   └─ 多场景模式：场景卡片列表，每个卡片包含（场景说明 + 出分玩家 + 条件 + 计分）
```

### 状态管理

- `isMultipleScenarios`：当前是否多场景模式（`boolean`）
  - `false`（默认）：单场景模式
  - `true`：多场景模式
- `formItems`：始终存储为数组（兼容原有数据结构）
  - 单场景：数组长度 = 1
  - 多场景：数组长度 > 1

### 交互流程

| 操作 | 当前状态 | 目标状态 | 动作 |
|------|----------|----------|------|
| 初始化（空数据） | - | 单场景 | 数组长度 = 1 |
| 初始化（已有数据） | - | 根据数组长度自动判断 | `length > 1` ➜ 多场景；`length = 1` ➜ 单场景 |
| 单场景下点击「添加场景」 | 单场景，长度 = 1 | 多场景，长度 = 2 | 插入新项，设置 `isMultipleScenarios = true` |
| 多场景下点击「添加场景」 | 多场景 | 保持多场景 | 插入新项 |
| 删除场景后，长度 > 1 | 多场景 | 保持多场景 | 删除项 |
| 删除场景后，长度 = 1 | 多场景 | 自动切回单场景 | 设置 `isMultipleScenarios = false` |

### UI变化

**单场景模式（单栏布局）**：
```
[出分玩家]
[计分表达式]
[添加场景] 按钮（始终显示，点击后切换多场景）
```

**多场景模式（卡片列表）**：
```
[场景 1] 卡片
  [场景说明]
  [出分玩家]
  [条件表达式] [计分表达式] (两栏)
[场景 2] 卡片
  ...
[添加场景] 按钮（在列表底部）
```

## 实现步骤

### 步骤 1：修改数据初始化逻辑
- 在 `initData()` 函数中，根据 `formItems.length` 自动设置 `isMultipleScenarios`
- 默认空数据时，保证 `formItems.length = 1`，`isMultipleScenarios = false`

### 步骤 2：重构模板结构
- 移除 `el-tabs` 标签页
- 使用 `v-if/v-else` 根据 `isMultipleScenarios` 分别渲染：
  - `v-if="!isMultipleScenarios"`：单场景布局
  - `v-else`：多场景卡片列表布局
- 统一在计分区域底部保留「添加场景」按钮

### 步骤 3：修改 `addFormItem` 函数
- 当前是单场景（`length = 1`）：
  - 添加新项后 `length = 2`
  - 设置 `isMultipleScenarios = true`
- 当前已是多场景：
  - 直接添加，不改变模式

### 步骤 4：修改 `removeFormItem` 函数
- 删除后检查 `formItems.length`
- 如果 `length <= 1`：设置 `isMultipleScenarios = false`

### 步骤 5：数据结构兼容性
- 单场景数据存储：使用 `formItems[0]`，字段包括 `mark`/`loser`/`calcExprText`/`judgeExprText`
- 输出数据时，保持原有的输出格式不变（`expr_list` 数组）
- 保证向后兼容，不破坏现有数据读取

### 步骤 6：样式调整
- 单场景：全宽布局，保持简洁
- 多场景：卡片列表，保持原有的渐变配色和圆角设计
- 添加平滑的过渡动画

## 预期效果

| 场景 | UI展示 | 用户体验 |
|------|--------|----------|
| 默认空配置 | 单场景，只有一行输入 | 简洁，大多数简单规则够用 |
| 需要多个规则 | 点一下「添加场景」自动切多场景 | 自然流畅，不需要手动切标签 |
| 删除到只剩一个 | 自动切回单场景 | 智能简化，界面更清爽 |
| 已有数据加载 | 根据项数自动判断模式 | 开箱即用，不需要用户记忆 |
