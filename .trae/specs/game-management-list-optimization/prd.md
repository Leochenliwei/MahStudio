# 游戏配置列表优化 - 产品需求文档

**版本**: v1.0
**日期**: 2026-03-25
**负责人**: 产品经理

---

## 1. 需求背景

当前游戏配置列表查询功能使用体验不佳，存在以下痛点：
- 无法快速精准定位单条数据
- 无法一键锁定自己创建的游戏
- 无法快速筛选最近新增的内容
- 交互体验缺失（无回车查询、无状态记忆）

## 2. 需求目标

提升游戏列表查询效率，覆盖 80% 日常查询场景

## 3. 功能清单

| 功能 | 描述 | 优先级 |
|-----|------|-------|
| 仅看我的状态缓存 | 勾选状态本地缓存，刷新后自动保留 | P0 |
| ID精准查询 | 支持输入游戏ID精准匹配 | P0 |
| 时间筛选 | 提供全部/今天/近7天/近30天/本月快捷选项 | P0 |
| 回车触发查询 | ID、名称输入框按回车自动查询 | P1 |
| 筛选状态记忆 | 所有筛选条件刷新后自动恢复 | P1 |
| 默认排序 | 按编辑时间倒序排列 | P2 |

## 4. UI 交互

### 布局
```
[仅看我的☑] │ [游戏ID] │ [游戏名称] │ [时间筛选▼] │ [查询] [重置]
```

### 字段说明
| 字段 | 类型 | 说明 |
|-----|------|------|
| 仅看我的 | 复选框 | 勾选后过滤当前用户创建/编辑的游戏 |
| 游戏ID | 输入框 | 精准匹配游戏ID |
| 游戏名称 | 输入框 | 模糊匹配游戏名称 |
| 时间筛选 | 下拉框 | 全部/今天/近7天/近30天/本月 |

## 5. 验收标准

- [ ] 仅看我的复选框状态在页面刷新后保持
- [ ] 时间筛选下拉框可正常选择并过滤数据
- [ ] ID/名称输入框按回车可触发查询
- [ ] 筛选条件在刷新后自动恢复
- [ ] 列表默认按编辑时间倒序排列

## 6. 技术方案

- **前端框架**: Vue 3 Composition API
- **状态持久化**: localStorage
- **样式**: 原生 CSS（保持项目一致性）

## 7. 影响范围

- 修改文件: `src/views/admin/GameManagement.vue`
- 无后端 API 变更
